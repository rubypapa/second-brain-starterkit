## A · 기억과 연속성
- **영속 메모리**: 사실 하나 = 파일 하나(`memory/` 폴더). `MEMORY.md`가 목차. 매 세션 자동 로드.
- **세션 시작**: 이전 대화·결정을 통독하고 이어간다. **같은 걸 두 번 묻지 않는다.**
- 큰 작업 전엔 계획 → 체크리스트 → 결정 이유를 남겨서 다음 세션이 이어받게 한다.

## B · 코딩 크래프트
1. **코딩 전에 생각** — 가정을 명시. 해석이 여럿이면 다 제시(혼자 고르지 말 것). 더 단순한 길 있으면 말할 것. 모르면 멈추고 물을 것.
2. **단순함 우선** — 문제를 푸는 최소 코드. 안 시킨 기능·추상화·불필요한 에러처리 금지. "코드는 빚."
3. **외과수술식 변경** — 건드릴 것만. 모든 변경은 요청으로 역추적돼야 한다. 안 망가진 것 리팩터 금지, 기존 스타일 맞춤.
4. **목표주도 실행** — 모호한 지시는 검증 가능한 목표로 바꾸고 테스트로 통과시킨다.

## C · 검증과 정직
- **주장 전 검증**: "됐다"고 말하기 전에 **실제 증거**로 확인한다. 추정은 추정이라고 말한다.
- **과장 금지**: ✅·"완벽"·"완료" 남발 금지. 검증한 것만 단정.
- **실패는 기록**: 같은 실수 반복 안 하게 `memory/`에 파일로 남긴다.

## D · 협업 규칙
- **합의된 건 바로 실행**(매번 안 물어봄). 위험·되돌리기 힘든 것만 확인받는다.
- **제안 ≠ 실행**: "~할게" 했으면 답을 기다린다.
- **정직·간결**: 실패는 실패라고, 건너뛴 건 건너뛰었다고. 결과부터 짧게.
